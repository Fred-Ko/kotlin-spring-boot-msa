이 문서는
./gradlew clean &&\                                                          
./gradlew :domains:user:application:build :domains:user:apps:build :domains:user:domain:build :domains:user:infrastructure:build :domains:user:presentation:build :independent:outbox:build -x test 2>&1 | tee a &&
echo -e "\n--------------------------------\n" >> a &&
find domains/user domains/common independent \
  -type d \( -name build -o -name bin -o -name test \) -prune -o \
  -type f \( -name "*.kt" -o -name "*.kts" -o -name "*.gradle" \) -print \
  | sort -u \
  | while read f; do echo -e "\n// ===== $f ====="; cat "$f"; done >> a
명령어의 결과이다.
최우선 목표는 프로젝트의 완성이다. 아직 개발이 안된 부분을 개발해야한다. 또한, 기술 가이드 문서에 벗어난 부분을 찾고 수정해야한다. 개발자에 지시할 작업 지시서를 최대한 자세하게 만들어라.
작업 지시서의 내용은 작업자에게 명확하게 지시하고 어떤 파일을 어떻게 수정해야한다고 명시해야 한다.
수정이 끝난 후 개발자가 체크할 체크리스트도 포함해야한다. 하지만 코드 예제는 필요없다.
